---
name: radar-invest
description: 投资研究备忘录（最近24h）。强调证据链、反证与可执行验证，不输出买卖建议。
---

执行方式：
- 直接执行完整工作流并输出最终 Markdown。
- 禁止输出确认问题、模板、执行过程、工具说明、占位符。
- 必须先调用 MCP 工具，再输出结论。
- 若没有满足标准的线索，明确输出“今日无高置信投资线索”。

来源规则：
- 使用 `config/source_registry.yaml` 作为来源级别基准。
- 主线索优先 `[C/]` / `[D/]`，`[M/]` 只做补充，不得单独支撑结论。
- 纯社会新闻、泛热搜、情绪型传播内容不得进入研究备忘录。

硬信号标准：
- 监管/政策明确动作、财报/指引、定价与价格战、供应链变化、产能扩张、合作落地、平台开放、重要新品对产业链的直接影响。

工作流：
1) `resolve_date_range`：expression="最近24小时" -> RANGE
2) `get_latest_news`：limit=80, include_url=true
3) `aggregate_news`：date_range=RANGE, limit=40
4) `get_trending_topics`：top_n=12, mode="daily"
5) 筛出 2~4 条事件驱动线索；若不足 2 条，可以少于 2 条，但不得硬凑
6) 对每条线索按需调用：
   - `analyze_topic_trend(topic, date_range=RANGE)`
   - `analyze_sentiment(topic, date_range=RANGE)`（仅作辅助）
7) 输出研究备忘录

输出要求：
- 每条线索都必须有：`来源`、`事实`、`影响链（推断）`、`关键变量`、`风险 / 反证`、`待验证清单`。
- `风险 / 反证` 至少 3 条；`待验证清单` 至少 3 条。
- `来源` 保留原始源名或域名，并尽量保留 `[C/]` / `[D/]` / `[M/]` 标签。
- 不允许给出“买 / 卖 / 仓位 / 目标价”等建议。

输出格式（Markdown）：
# 研究备忘录（最近24h，非投资建议）
## 1) 今日最值得跟踪的 2~4 条线索
### 线索A：<具体事件 / 公司 / 政策>
- 来源：
- 事实（3~6条）：
- 影响链（推断，写成因果链条）：
- 关键变量（3~5个）：
- 风险 / 反证（至少3条）：
- 待验证清单（至少3条）：

## 2) Watchlist（仅列名 + 触发条件）
- <公司 / 赛道>：关注原因；触发条件

兜底格式：
# 研究备忘录（最近24h，非投资建议）
今日无高置信投资线索。高价值事件不足，不输出低质量推断。
