---
name: radar-router
description: 今日总报（最近24h）。一次取数，输出投资/产品/论文三段总览；用于自动化主入口。
---

执行方式：
- 直接执行完整工作流并输出最终 Markdown。
- 禁止输出确认问题、模板、占位符、执行过程、工具调用说明。
- 必须先调用 MCP 工具，再输出任何结论。
- 若证据不足，必须明确写“今日无足够强信号”，不要用弱相关内容凑数。

来源规则：
- 使用 `config/source_registry.yaml` 作为来源级别基准。
- `[C/]` 和 `[D/]` 才能决定主结论；`[E/]` 只能补“为什么重要”；`[M/]` 只能做线索补充。
- 仅来自热榜平台、且没有 `[C/]` / `[D/]` / `[E/]` 补证的内容，默认视为弱信号。

硬信号标准：
- 新品发布、重大版本更新、SDK/API 开放、定价变化、合作落地、监管/政策明确动作、财报/指引/供应链变化、真实 benchmark / 论文 / 开源 release。

默认关注域：
AI / 大模型 / 芯片 / 算力 / 端侧 / agent / 机器人 / 自动驾驶 / 智能硬件 / 智能座舱 / 生产力工具 / 消费电子

工作流：
1) `resolve_date_range`：expression="最近24小时" -> RANGE
2) `get_latest_news`：limit=60, include_url=true
3) `aggregate_news`：date_range=RANGE, limit=30
4) `get_trending_topics`：top_n=12, mode="daily"
5) 从新闻、聚合、话题中筛出 5~8 条强信号；总数不足时允许少于 5 条
6) 对入选主题按需调用：
   - `analyze_topic_trend(topic, date_range=RANGE)`
   - `analyze_sentiment(topic, date_range=RANGE)`（仅作辅助）
7) 输出最终日报

输出要求：
- 总计 5~8 条重点；每段最多 3 条。
- 每条必须包含：`来源`、`事实`、`为什么重要`、`待验证 / 下一步`。
- `来源` 保留原始源名或域名，并尽量保留 `[C/]` / `[D/]` / `[E/]` / `[M/]` 标签。
- 不要写“基于技能”“我已执行”“以下是模板”“如需我继续”等话术。

输出格式（Markdown）：
# 今日总报（最近24h）
## 1) 投资线索（非投资建议）
- 线索1：<具体事件 / 公司动作>
  - 来源：
  - 事实：
  - 为什么重要：
  - 风险 / 反证：
  - 待验证 / 下一步：

## 2) 产品机会
- 机会1：<具体产品 / 功能 / 平台动作>
  - 来源：
  - 事实：
  - 为什么重要：
  - 最小验证 / 下一步：

## 3) 论文与技术
- 方向1：<具体论文 / benchmark / 工程更新>
  - 来源：
  - 事实：
  - 为什么重要：
  - 边界：
  - 待验证 / 下一步：

兜底格式：
# 今日总报（最近24h）
今日无足够强信号。仅保留高置信度来源后，未发现值得进入日报头条的事件。
