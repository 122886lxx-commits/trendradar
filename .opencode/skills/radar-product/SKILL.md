---
name: radar-new-products
description: 新品雷达（最近24h）。优先公司官方源筛出 AI 新品 / 大更新 / 新服务，并给出定位、人群、技术、痛点与生态位解读。
---

执行方式：
- 直接执行完整工作流并输出最终 Markdown。
- 禁止输出确认问题、模板、执行过程、占位符。
- 必须先调用 MCP 工具，再输出结论。
- 如果没有满足标准的条目，明确输出“今日无满足标准的新品或重大更新”。

来源规则：
- 使用 `config/source_registry.yaml` 作为来源级别基准。
- `[C/]` / `[D/]` 为主，决定是否入选。
- `[E/]` 只补背景，不决定入选。
- `[M/]` 只能作为发现线索，最多 2 条，且必须明确标注“媒体”或“传闻”。

关注范围：
- 海外：OpenAI / Google / DeepMind / Anthropic / Meta / xAI
- 国内：字节跳动 / 阿里 / 腾讯 / DeepSeek / MiniMax / Kimi / 智谱
- 初创：允许进入，但必须有明确发布动作与可核验事实

硬信号标准：
- 新品发布、重大版本更新、SDK/API 开放、服务正式上线 / 公测、价格方案更新、硬件曝光、合作落地。

工作流：
1) `resolve_date_range`：expression="最近24小时" -> RANGE
2) `get_latest_news`：limit=160, include_url=true
3) `get_trending_topics`：top_n=30, mode="daily"
4) `aggregate_news`：date_range=RANGE, limit=60
5) 从新闻 / 话题 / 聚合中抽取候选：主体 + 产品/功能 + 发布动作
6) 优先保留 `[C/]` / `[D/]`；媒体线索最多保留 2 条
7) 对每个入选新品按需调用：
   - `analyze_topic_trend(topic, date_range=RANGE)`
8) 输出新品清单，最多 5 条；证据不足时宁缺毋滥

输出要求：
- 每条必须含：`信号等级`、`来源`、`证据（事实）`、`定位 / 目标人群`、`技术与形态`、`解决的痛点`、`生态位与竞品对照`、`7天最小验证`、`待验证清单`。
- `来源` 保留原始源名或域名，并尽量保留 `[C/]` / `[D/]` / `[M/]` 标签。
- 媒体 / 传闻条目必须显式降级，不做过度推断。

输出格式（Markdown）：
# 新品雷达（最近24h）

> 说明：新品条目优先来自公司官方源或开发者源；媒体仅用于线索发现。

## 新品1：<产品名> — <一句话定位>
- 信号等级：官宣 / 准官宣 / 媒体 / 传闻
- 主体（公司 / 团队）：
- 来源（保留原始源名或域名）：
- 证据（事实，3~6条）：
- 定位 / 目标人群：
- 场景与核心任务：
- 技术与形态：
- 解决的痛点：
- 生态位与竞品对照：
- 关键指标：
- 7天最小验证：
- 待验证清单（至少3条）：

兜底格式：
# 新品雷达（最近24h）
今日无满足标准的新品或重大更新。仅保留高置信度来源后，未发现足够强的产品信号。
